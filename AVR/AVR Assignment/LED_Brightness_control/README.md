

# LED Brightness Control using Timer0 PWM
 
## Objective

This project demonstrates control of LED brightness  using hardware PWM generated by **Timer0** of Atmega328P. The duty cycle is changed using a push button.

## Features

-   Uses **Timer0 Fast PWM mode**
-   PWM output on **OC0A (PD6)**
-   Starts at **25% duty cycle**
-   Push button cycles through:
    -   25%
    -   50%
    -   75%
    -   100%
    -   OFF
-   Debounced button input using software delay + edge detection
-   Clean state-based duty cycle control using lookup table
- 
## Hardware Requirements

-   AVR Microcontroller (ATmega328P)
-   LED 
-   Push button
-   Pull-up resistor (internal pull-up used in code)

## Pin Configuration

| Signal | Pin | Description |
| ----------------- | --- | --------------------------- |
| PWM Output (OC0A) | PD6 | Controls LED brightness |
| Push Button | PD2 | Input with internal pull-up |

## PWM Configuration
| Parameter | Value |
| -------------- | ------------- |
| Mode | Fast PWM |
| Output Mode | Non-inverting |
| Prescaler | 64 |
| PWM Frequency | ~976 Hz (≈1 kHz) |

## Duty Cycle Mapping
| Duty Cycle | OCR0A Value |
| ---------- | ----------- |
| 25% | 63 |
| 50% | 127 |
| 75% | 191 |
| 100% | 255 |
| OFF | 0 |

## Working Principle

-   Timer0 generates a PWM signal on OC0A pin.
-   Duty cycle is controlled by writing values to `OCR0A`.
-   A push button triggers an interrupt-like behavior using edge detection in software.
-   Each valid press updates the duty cycle in sequence.
-   Debouncing ensures stable input by adding delay and rechecking button state.

## Code Structure

-   `Init_FastPWM()`  
    Configures Timer0 in Fast PWM mode and sets initial duty cycle.
-   `Init_pins()`  
    Configures PWM output pin and button input pin with pull-up.
-   `Update_brightness()`  
    Updates OCR0A value based on current state.
-   `main()`  
    Handles button press detection and debouncing logic.
