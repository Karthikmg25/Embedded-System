


# Timer0 PWM Level Control 

## Objective

This project demonstrates LED brightness control using hardware PWM generated by **Timer0 in Fast PWM mode**. A push button is used to cycle through predefined duty cycle levels.

---

## Features

- Timer0 Fast PWM mode
- PWM output on OC0A (PD6)
- Starts at 25% duty cycle
- Button cycles through:
  - 25%
  - 50%
  - 75%
  - 100%
  - OFF (0%)
- Software debouncing using delay + edge detection
- State-based PWM control using lookup table

---

## Hardware Requirements

- AVR Microcontroller (ATmega328P or similar)
- LED 
- Push button
- Pull-up resistor (internal pull-up used)


---

## Pin Configuration

| Signal | Pin | Description |
|--------|-----|-------------|
| PWM Output (OC0A) | PD6 | LED brightness / motor control |
| Push Button | PD2 | Input with internal pull-up |

---

## PWM Configuration

| Parameter | Value |
|----------|------|
| Mode | Fast PWM |
| Output Mode | Non-inverting |
| Prescaler | 64 |
| PWM Frequency | ~976 Hz |

---

## Duty Cycle Mapping

| Duty Cycle | OCR0A Value |
|------------|-------------|
| 25% | 63 |
| 50% | 127 |
| 75% | 191 |
| 100% | 255 |
| OFF | 0 |

---

## Working Principle

- Timer0 generates PWM signal on OC0A (PD6)
- Duty cycle is controlled using OCR0A register
- Push button cycles through predefined PWM levels
- Each valid press updates the duty cycle
- Debouncing ensures stable button detection

---

## Code Structure

### Init_FastPWM()

Configures Timer0 in Fast PWM mode, non-inverting output, and prescaler 64.

### Init_pins()

Sets PD6 as output and PD2 as input with pull-up enabled.

### Update_PWM_Level()

Updates OCR0A based on current state and cycles through PWM levels.

### main()

Handles button press detection using edge detection and debouncing logic.

---

## Button Logic

- Falling edge detection used (HIGH → LOW transition)
- 20 ms delay used for debouncing
- Ensures single update per press
